## Before Pushing to GitHub - Final Checks

### 1. Verify .env files are ignored
```bash
git check-ignore backend/.env frontend/.env
```
Should output:
```
backend/.env
frontend/.env
```

### 2. Check what will be committed
```bash
git status
```
Should NOT show any `.env` files

### 3. Search for any remaining secrets
```bash
# Search for potential API keys (should only find .env.example)
git grep -i "api.*key.*=" | grep -v ".env.example" | grep -v "your_"
```

### 4. Review staged changes
```bash
git diff --cached
```

## Emergency: If You Accidentally Committed Secrets

If you already pushed secrets to GitHub:

1. **Immediately rotate all API keys** (get new ones)
2. Remove from Git history:
```bash
git filter-branch --force --index-filter \
  "git rm --cached --ignore-unmatch backend/.env" \
  --prune-empty --tag-name-filter cat -- --all

git push origin --force --all
```
3. Or use BFG Repo-Cleaner (easier): https://rtyley.github.io/bfg-repo-cleaner/
